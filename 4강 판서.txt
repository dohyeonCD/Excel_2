@ VBA @
1. ms 오피스 데이터를 가지고 분석하기에는 제일 적합. ms만 분석할 수 있음. 
2. 데이터를 구조화, 자동화할 때 사용할 수 있는 언어.  



@ 코딩용 폰트 @
코딩전용 글꼴: 나눔고딕코딩 (1, l,ㅣ, o, 0 비교 잘되라구 네이버가 배포함~)



@ VBE 세팅 @
VBE 창으로 들어가기(VBA 편집할 수 있는 창) 단축키: alt + f11
리본 보기 메뉴-> '속성 창' 열기
리본 보기 메뉴-> '코드' 열기
리본 보기 메뉴-> '도구 모음'-> 편집 툴바 추가
리본 도구 메뉴-> '옵션'-> 편집기 탭에서 변수 선언 요구 체크-> 편집기 형식 탭에서 글꼴 변경(나눔고딕코딩Bold(한글))



@ VBE 기본 @
가독성 높이기: tab으로 들여쓰기
매크로(프로시저) 실행 단축키: f5
순서대로 실행 단축키: f8           (노란 줄 위까지 실행한 것임)
주석: 맨 앞에 ' 추가.




@ 엑셀 개체모델(족보) @
(application(엑셀)) > (workbooks(통합문서) 하나만 사용할 때 생략가능) > (work)sheets(워크시트) > range(셀) 이런 식으로 입력해야 함.
range 대신 cells(행,열)도 됨.
cells(행과 열 숫자. 혹은 특정 행, 열 지정하려면 문자로. "az")
	숫자로만 지정해야 할 때가 있음. 그럴 때는 ctrl + g 해서 직접실행창 열고-> ? range("az1").Column 하면 az1셀의 column 숫자를 알려줌.


<<프로시저(매크로) 작성>>
1. 모듈 생성
VBAProject 선택-> M.S-> 삽입-> '모듈'

2. 족보순으로.
workbooks 쓸 때 항상 파일 형식도 같이 써야 함. ex) 개체모델.xlsm        xlsm : 매크로 통합.   xlsx : 일반 통합
worksheets 순서로 지정하려면 숫자만 입력.
시트의 위치, 이름이 바뀌어도 오류가 안 나도록-> VBA가 인지하는 시트 이름으로 지정(Sheet1). VBA 스스로, 시트가 만들어진 순서대로 이름 만들어놓음.


<<실행 버튼 (RUN) 만들기>>
1. 아무 셀-> 리본 개발 도구 메뉴-> '삽입'-> 양식 컨트롤-> '단추'
2. 어떤 프로시저 연결할 건지 지정-> 매크로 위치 지정(맨 밑 문서)
(예쁜 단추 만들기 : 리본 삽입 메뉴-> '도형'-> 도형 M.R-> '매크로 지정')



@ 변수 종류 @           
1. 일반 변수
  - 숫자, 문자, 날짜, 논리값과 같은 값들을 변수로 받는 것. 
  - Msgbox로 일반변수에 들어온 값 검증
  - f8로 실행하면서 변수명에 마우스 갖다 대면 보이는 값으로 검증

2. 개체 변수
  - 액셀의 개체를 변수로 받는 것. (range~)
  - 개체변수는 변수 지정 시, 반드시 set 문을 붙여야 함.
  - 개체.select로 검증 (그 범위를 잡는지 확인)
  - 조사식 창으로 개체변수에 들어온 값 검증 (리본 보기 메뉴-> 조사식 창-> 변수를 창에다 퐁당~하고 end sub 앞에 중단점 설정(마우스로 빈 공간 클릭)-> 
    실행-> Value2를 열어서 셀 주소와 데이터 확인)
